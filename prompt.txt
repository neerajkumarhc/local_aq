Create an HTML/JavaScript webpage for deployment on Cloudflare that displays real-time PM2.5 air quality data based on user location. The page should:

CORE FUNCTIONALITY:
- Use the browser's Geolocation API to get the user's coordinates
- Query the OpenAQ API (latest version) to find the nearest active PM2.5 monitoring station
- Display the current PM2.5 level and 24-hour average PM2.5 level from that station
- Calculate cigarette equivalents based on the 24-hour average (use standard conversion: 1 cigarette ≈ 22 μg/m³ PM2.5)

API KEY MANAGEMENT:
- The OpenAQ API key will be stored as a Cloudflare secret variable (not hardcoded in the HTML)
- Create a simple Cloudflare Worker or serverless function that:
  * Accepts the user's coordinates as query parameters
  * Retrieves the API key from Cloudflare environment variables
  * Makes the OpenAQ API request server-side (keeping the API key secure)
  * Returns the station data to the client
- The HTML/JavaScript client calls this worker endpoint instead of directly calling OpenAQ API
- Implement CORS headers appropriately in the worker to allow requests only from your domain

CACHING & RATE LIMITING:
- Implement browser-based caching using IndexedDB or sessionStorage to prevent excessive API calls
- Cache strategy:
  * Store API responses with timestamps
  * Do not make a new API request if a valid cache entry exists for the same station within the last 30 minutes
  * If the user's location changes significantly (more than ~5 km away), query for a new nearest station and cache that result
  * Clear cache entries older than 2 hours automatically
  * Display a cache indicator showing when data was last updated and if it's from cache
- Add a manual "Refresh Now" button that allows users to force a fresh API call, but implement a cooldown (e.g., minimum 5 minutes between manual refreshes) to prevent rapid-fire requests
- Log cache hits vs. API calls to help monitor usage patterns

EXPOSURE GUIDANCE:
- Implement the following PM2.5 exposure guidance framework with these bands:
  * Ideal: 0-5 μg/m³ - No special measure needed
  * Acceptable: >5-15 μg/m³ - Home: HEPA purifier | Work: Standard filtration
  * Caution: >15-25 μg/m³ - Home: High-capacity purifier with IAQ monitoring | Work: High-efficiency filtration
  * Elevated Caution: >25-37.5 μg/m³ - Home: High-efficiency mechanical ventilation | Work: High-efficiency filtration
  * High: >37.5-50 μg/m³ - Home: High-efficiency mechanical ventilation | Work: High-efficiency filtration | Outdoor: Respirator
  * Very High: >50-75 μg/m³ - Home: High-efficiency mechanical ventilation | Work: High-efficiency filtration | Transit: Mask | Outdoor: Respirator
  * Extreme: >75 μg/m³ - Ideal exposure cannot be met. Home: High-efficiency mechanical ventilation | Work: High-efficiency filtration | Outdoor: Respirator

- For each band, show:
  1. Current exposure level classification
  2. Recommended measures to achieve "Acceptable" exposure (or "Ideal" if already in that range)
  3. Calculate projected daily exposure with recommended measures applied using these reduction factors:
     - No special measure: 1.00 (0% reduction)
     - Doors and windows closed: 0.80 (20% reduction)
     - HEPA air purifier: 0.50 (50% reduction)
     - High-capacity purifier with IAQ monitoring: 0.25 (75% reduction)
     - High-efficiency mechanical ventilation filtration: 0.05 (95% reduction)
     - Standard filtration at work: 0.60 (40% reduction)
     - High-efficiency filtration at work: 0.10 (90% reduction)
     - Baseline transit: 0.50 (50% reduction)
     - Mask in transit: 0.30 (70% reduction)
     - Respirator: 0.05 (95% reduction)
  4. Use time allocation: 13.5 hours home, 8 hours work/school, 1.5 hours transit, 1 hour outdoor
  5. Calculate and display the resulting daily average exposure with recommended measures
